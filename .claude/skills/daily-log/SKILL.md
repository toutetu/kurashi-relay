# 日報記録

くらしリレー開発の日報を、作業日ごとに1本ずつ仕上げて保存する。

## 前提: 下書きは自動生成されている

毎日20時に Windows タスク `KurashiRelay-DailyLogDraft` が
`scripts/generate-draft.ps1` を実行し、前回日報以降の作業日について
`docs/logs/daily_log_YYYY-MM-DD.md` の下書きを生成している。
下書きでは、履歴から書ける「作業内容・感想」はCursorが清書済みで、
人にしか分からない **作業時間・体調・感想の出来事・次回予定は【要確認】** で残っている。

このスキルの主な仕事は、その【要確認】を本人に聞いて埋め、レビューして確定させること。
トークン節約のため、フォーマット全文の読み込みや本文の書き直しはCursorに任せ、
Fableは「質問・レビュー・確定コミット」に集中する。

## 手順

### 1. 仕上げ対象の下書きを探す

`docs/logs/daily_log_*.md` のうち、本文に【要確認】を含むファイルを新しい順に列挙する:

```
git status --short docs/logs/; grep -l "【要確認】" docs/logs/daily_log_*.md
```

- 下書きが見つかった → 手順2へ(仕上げモード)
- 一つも無い(自動生成が走っていない等) → 手順5へ(生成モード)

### 2. 本人に質問する(スキップ禁止)

見つかった下書きの各日について、【要確認】になっている項目をまとめてチャットで質問する。
禁止事項「メモにないことを書かない」を守るためのステップなので省略しない。

1. その日の作業時間(○時間)
2. 体調スコア: フィジカル / メンタル(各10点満点)
3. 感想に入れたい出来事(面談・レビュー・生活のこと等。なければ「なし」)
4. 次回の作業予定(下書きのコミット内容から案を出して確認する形でよい)

回答が得られない項目は創作せず【要確認】のまま残す。

### 3. 下書きを更新する

回答をもとに【要確認】を置き換える。特に作業時間まわり:

- 累計作業時間 = 前回日報の累計 + その日の作業時間。のこり = 100時間 − 累計
- 前回日報の累計が【要確認】のままだと、その日以降ずっと累計が確定しない。
  古い日から順に作業時間を確定させると累計が積み上がる
- 数値記入だけなら軽微なのでFableが直接編集してよい。
  本文の書き直しが必要なら手順5と同じくCursorにメモ経由で委譲する

### 4. レビューして確定

- 見出し構成がテンプレどおりか / 累計・のこりの計算が合っているか /
  固有名詞(テーブル名・ライブラリ名)を使っていないか / 【要確認】が残っていないか を確認
- 確定した日報をチャットにも提示し、
  `docs(logs): add daily log for YYYY-MM-DD` の形式で1日1コミットして push する
  (コミット・push は確認不要の恒常ルール)

### 5. 生成モード(下書きが無いとき)

自動生成が走っていない場合は、その場でスクリプトを実行して下書きを作ってから手順2へ戻る:

```
powershell -ExecutionPolicy Bypass -File .claude/skills/daily-log/scripts/generate-draft.ps1
```

このスクリプトが前回日報以降の作業日を判定し、gitから材料を集めてメモ
(`docs/logs/_nippo_memo.md`)を書き、Cursorに清書させる。
Cursorが使えない環境なら、`-NoCursor` でメモだけ作り、
`docs/logs/日報AI代筆フォーマット.md` に従ってFableが直接清書してもよい。

## 関連ファイル

- 空欄テンプレート: `docs/logs/日報テンプレート.md`
- 自動生成スクリプト: `.claude/skills/daily-log/scripts/generate-draft.ps1`
