# 日報記録

くらしリレー開発の日報を、前回の続きから作業日ごとに1本ずつ作成して保存する。

トークン節約のため、本文の執筆は Cursor(composer-2.5)に委譲する。
Fable の担当は「日付の特定・ユーザーへの質問・委譲メモ作成・軽いレビュー」のみ。
フォーマット全文の読み込みと本文執筆は Cursor 側にやらせる(恒常ルールの委譲体制どおり)。
Cursor が使えない・失敗した場合のみ、Fable が直接
`docs/logs/日報AI代筆フォーマット.md` を読んで執筆してよい。

## 手順

### 1. 前回の日報を特定する

`docs/logs/daily_log_YYYY-MM-DD.md` のうち最新日付のファイルを探し、
「累計作業時間」の行だけを Grep で読み取る(全文は読まない)。

### 2. 対象の作業日を特定する

前回日報の翌日から依頼日までの間で、作業があった日を列挙する:

```
git log --all --since="<前回日報の翌日> 00:00" --date=format:"%Y-%m-%d" --pretty=format:"%ad %s"
```

コミットがない日にも作業(調査・打ち合わせ等)をしていた可能性があるため、
列挙した作業日をユーザーに提示し「この日で合っていますか? 他に作業した日はありますか?」と確認する。

### 3. ユーザーに質問する(スキップ禁止)

履歴からは分からない以下を、作業日ごとにまとめてチャットで質問する。
フォーマットの禁止事項「メモにないことを書かない」を守るためのステップなので、省略しない。

1. その日の作業時間(○時間)
2. 体調スコア: フィジカル / メンタル(各10点満点)
3. 感想に入れたい出来事(面談・レビュー・生活のこと等。なければ「なし」でOK)
4. 次回の作業予定(こちらから案を出して確認する形でもよい)

回答が得られなかった項目は【要確認】としてメモに書く(創作しない)。

### 4. 委譲メモを書く

`docs/logs/_nippo_memo.md` に、Cursor への指示と日ごとのメモを書く。
指示を本体プロンプトに書かないのは、cursor-agent の .cmd ラッパーが
複数行プロンプトを最初の改行で切ってしまうため(実測済みの落とし穴)。

```markdown
# 日報執筆の指示

docs/logs/日報AI代筆フォーマット.md を読み、その構成・文体・粒度・禁止事項・文例に
厳密に従って、以下の各日の日報を作成してください。

- 保存先: docs/logs/daily_log_YYYY-MM-DD.md(1日1ファイル)
- ファイルには日報本文のみを書く(前後に説明文をつけない)
- 累計作業時間 = 前回累計 + その日の作業時間。のこり = 100時間 − 累計。
  複数日あるときは日付順に累計を積み上げる
- メモにない作業・感想を創作しない。【要確認】はそのまま残す
- Phase 構成と進捗の表は、状態が変わった日だけ入れる

## YYYY-MM-DD(曜)のメモ
- 前回の累計作業時間: XX時間
- 今日の作業時間: X時間
- やったこと: (コミット履歴・会話から。箇条書き)
- 体調: フィジカル X / メンタル X
- 感想に入れたい出来事: (ユーザーの回答)
- 次回の作業予定: (箇条書き2〜3個)
```

### 5. Cursor に執筆を委譲する

Bash で実行する。サンドボックス解除(`dangerouslyDisableSandbox: true`)が必須
(api2.cursor.sh への通信がサンドボックス内だと ENOTFOUND になる)。
`run_in_background: true` で起動し、完了通知を待つ。

```
"$LOCALAPPDATA/cursor-agent/cursor-agent.cmd" -p --force --trust --model composer-2.5 --workspace . "docs/logs/_nippo_memo.md を読んで、書かれている指示を実行してください"
```

### 6. レビューして仕上げる

- 生成された `daily_log_*.md` を読み(短いので全文でよい)、次を満たすか確認する:
  見出し構成がテンプレどおりか / 累計・のこりの計算が合っているか /
  メモにないことを書いていないか / 固有名詞(テーブル名・ライブラリ名等)を使っていないか /
  【要確認】が勝手に埋まっていないか
- ずれが軽微なら Fable が直接修正、大きければメモを直して再委譲
- `docs/logs/_nippo_memo.md` を削除する(コミットしない)
- 完成した日報をチャットにも提示し、
  `docs(logs): add daily log for YYYY-MM-DD` の形式でコミットし push する
  (コミット・push は確認不要の恒常ルールに従う)
