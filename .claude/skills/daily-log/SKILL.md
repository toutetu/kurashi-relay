# 日報記録

くらしリレー開発の日報を、前回の続きから作業日ごとに1本ずつ作成して保存する。

## 書き方の正(毎回必ず読む)

日報を書く前に必ず `docs/logs/日報AI代筆フォーマット.md` を読み、
構成・文体・粒度・禁止事項・文例に従うこと。特に重要な点:

- 読み手は毎日コードを見ていない人。「何がどこまで進んだか」が伝わる粒度で書き、
  テーブル名・コンポーネント名・ライブラリ名などの固有名詞は原則使わない
- 材料(ユーザーの回答・コミット履歴・会話履歴)にないことを書かない。
  あいまいな点は勝手に埋めず【要確認】として残す
- 日報ファイルには本文のみを書く(前後に説明文をつけない)

空欄テンプレート: `docs/logs/日報テンプレート.md`

## 手順

### 1. 前回の日報を特定する

`docs/logs/daily_log_YYYY-MM-DD.md` のうち最新日付のファイルを探して読む。
そこから「累計作業時間」を読み取る(累計の計算に使う)。

### 2. 対象の作業日を特定する

前回日報の翌日から依頼日までの間で、作業があった日を列挙する:

```
git log --all --since="<前回日報の翌日> 00:00" --date=format:"%Y-%m-%d" --pretty=format:"%ad %s"
```

コミットがない日にも作業(調査・打ち合わせ等)をしていた可能性があるため、
列挙した作業日をユーザーに提示し「この日で合っていますか? 他に作業した日はありますか?」と確認する。

### 3. 各作業日の材料を集める

- その日のコミット内容と docs の変更から「今日の作業内容」の材料を作る
- 現在のセッションで作業していた日なら、会話の内容も材料にしてよい

### 4. ユーザーに質問する(スキップ禁止)

履歴からは分からない以下を、作業日ごとにまとめてチャットで質問する。
禁止事項「メモにないことを書かない」を守るためのステップなので、省略しない。

1. その日の作業時間(○時間)
2. 体調スコア: フィジカル / メンタル(各10点満点)
3. 感想に入れたい出来事(面談・レビュー・生活のこと等。なければ「なし」でOK)
4. 次回の作業予定(こちらから案を出して確認する形でもよい)

回答が得られなかった項目は創作せず【要確認】と書く。

### 5. 日報を書いて保存する

- 保存先: `docs/logs/daily_log_YYYY-MM-DD.md`(1日1ファイル・作業日ごとに1本)
- 累計作業時間 = 前回の累計 + その日の作業時間。のこり = 100時間 − 累計。
  複数日ぶんを書くときは日付順に累計を積み上げる
- 「Phase 構成と進捗」の表は、Phaseの状態が変わった日だけ入れる

### 6. 提示してコミット

作成した日報をチャットにも提示して修正依頼を受けられるようにしつつ、
`docs(logs): add daily log for YYYY-MM-DD` の形式でコミットし push する
(コミット・pushは確認不要の恒常ルールに従う)。
