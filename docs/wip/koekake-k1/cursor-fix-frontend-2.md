## 残1: Undo中/再取得前に古い推奨文を送れてしまう(前回指摘1が不十分)

現状: cancel成功後に一覧を invalidate しているが `void` で破棄し**再取得完了を待たない**。かつ取消中も
カードの「声かけ済み」ボタンが押せるため、再取得前に押すと古い level の文が送信本文になる
(`queries.ts:169`/`229`、`KoekakePage.tsx:103`/`120`)。

**修正(いずれかで確実に。推奨は両方)**:
1. cancel フローで invalidate を **await** する(`await queryClient.invalidateQueries({ queryKey: listKey })`。
   詳細キーも同様)。`void` 破棄をやめる。
2. **取消中および再取得完了までは、そのタスクの「声かけ済み」主ボタン(と詳細シートの声かけ操作)を
   disabled にする**(「更新中…」等の中立表示)。タスク単位の pending 状態で制御する。
   → これにより、再取得でカードの `suggested_prompt` がサーバ値へ確定するまで新規送信が起きない。
- 補強として、送信本文はカードの props キャプチャではなく **送信時点の最新キャッシュ**
  (`queryClient.getQueryData` の当該タスク `suggested_prompt`)から作ると、なお堅い(可能なら実施)。

## 残2: Undoトーストのタイマーが取消通信中も進み、失敗時に再試行できない(前回指摘4が不十分)

現状: Undoトーストの10秒自動消滅タイマーは DELETE 通信中も動き続け、遅い/期限直前の押下では通信中に
トーストが消える。その後 DELETE が失敗しても `undoError` の表示先がなく再試行できない
(`KoekakePage.tsx:69`/`129`)。

**修正**:
- **Undoボタン押下時に10秒自動消滅タイマーを即停止**し、トーストを「取り消し中…」のローディング状態で
  **DELETE解決まで表示し続ける**。
- 成功時にトーストを閉じる。**失敗時はトーストを残し、エラー文言 + [再試行] ボタン**を出す
  (再試行で再度 DELETE)。
- 10秒自動消滅は「まだUndoを押していない」間だけ有効。

## テスト追加(前回の不足を埋める・Codex指摘)

`KoekakePage.test.tsx` に追加:
