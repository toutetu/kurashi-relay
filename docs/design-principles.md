# くらしリレーの設計原則

1. くらしリレーは、母や娘を監視・評価するアプリではない。
2. 母へ集中した支援、待機、拘束、回復を可視化し、役割分担につなげる。
3. 娘本人の発言、母の確認、母の観察、母の推測を区別する。
4. 達成・失敗や頑張り不足という評価を行わない。
5. 日常で最も頻繁に使う操作を画面上部へ置く。
6. ホームでは詳細情報より、すばやい記録と今日の概要を優先する。
7. PCは一覧性と情報密度を優先する。
8. タブレットは日常の主入力端末として扱う。
9. スマートフォンは外出中の即時記録を優先する。
10. 小さい画面ではタブによる段階的開示を使用する。
11. 同じ機能を端末別に複製せず、同じコンポーネントを再配置する。
12. ホームは概要を表示し、詳細は専用画面へ分ける。
13. 主要操作は1〜2タップ、操作領域は44px以上とする。
14. 色だけで状態を表現しない。
15. 操作には押下、処理中、成功、失敗、未保存のフィードバックを返す。
16. 共通コンポーネントとデザイントークンを優先する。
17. 直接カラーコードや画面固有の重複スタイルを増やさない。
18. 200％表示でも操作不能や横スクロールを発生させない。
19. アクセシビリティと可読性を装飾より優先する。
20. 実データや子どものセンシティブ情報を不用意に共有しない。

## ホームと画面分割

- ホームには頻繁に使用する操作と最小限の今日情報だけを置く。
- 娘の情報は、母のホームへ混在させず専用画面へ分ける。
- 娘本人の入力と母の確認・観察を区別する。
- 予定への影響、対応事項、ゲーム進捗などの詳細情報はまとめ画面へ置く。
- PCホームは2段・3列の一覧性を優先する。
- スマホ・タブレットのホームは「記録」「今日」の2タブとする。
- ラストウォーは生活支援や予定への影響とは性質が異なるため、独立ページで扱う。
- 外部向けレポートや支援者向け資料へラストウォーの詳細を自動的に含めない。
- ゲーム進捗は本人の生活の一部として扱うが、支援評価や就労評価と混同しない。
- 色は確定値ではなく、共通トークンから後で変更できる構造にする。
- 画面固有の直接カラーコードを増やさない。
- 同じ段のカードは高さをそろえ、一覧性を高める。
- 固定高で内容を切らず、段の中で自然に高さを合わせる。
- カードの影は装飾を強調しすぎず、薄く右下方向へ使用する。
- 可読性と情報の区切りは、濃い影ではなく余白、枠線、背景、アクセントで表現する。
- クリック可能な要素は、hover、active、focusの各状態を一貫して表現する。
- hoverでは色と位置を変化させ、クリック可能であることを伝える。
- activeでは少し沈ませ、操作を受け付けた感触を返す。
- タッチ端末ではhoverに依存しない。
- 色の状態変化もデザイントークンで管理する。
