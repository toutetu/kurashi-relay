# 1. 設計方針

## 1-1. 役割分担

- 娘用画面は「見通しを決める場所」とする。
- 母用画面は「声かけを思い出し、記録する場所」とする。
- 日常行動中、娘にスマートフォン操作を要求しない。
- アプリは母の代わりに娘を監督するのではなく、母の記憶と記録を支援する。

## 1-2. 画面設計の優先順位

1. 母が今すべき声かけを一目で分かる
2. 1タップで声かけを記録できる
3. 同じタスクに何回声をかけたか分かる
4. 後から時刻と内容を確認できる
5. 娘用画面は項目数を絞る

# 4. 娘用画面設計

## 4-1. D-01 娘用ホーム

### 目的

帰宅後に、翌日までの見通しを短時間で決める。

### レイアウト

```text
┌────────────────────┐
│ 🐰 おかえり！        7/15（火） │
├────────────────────┤
│ ❤️ 今日何をする？              ＞ │
│ 🎒 明日何がいる？              ＞ │
│ 🕒 明日何時間目から登校？      ＞ │
├────────────────────┤
│ 🔄 今日の振り返り（19時（夕食後）ごろ） ＞ │
│ ⭐ メモ・思いついたこと        ＞ │
└────────────────────┘
```

### 表示ルール

- 主な項目は三つに固定する。
- 未完了件数を赤字で表示しない。
- 「母と決める」「今は決めない」を常に選べる。
- 娘が選択した結果は母用リマインダー生成に利用する。

## 4-2. D-02 今日すること

### 初期候補

- 宿題
- テスト勉強
- 学校の課題
- 遊ぶ
- 休む
- 入浴
- その他

複数選択を可能にする。時間指定は任意。

## 4-3. D-03 明日必要な物

### 入力方法

- 時間割から候補を表示
- よく使う物を表示
- 音声入力
- 写真メモ
- 母と確認

### 初期候補

- 宿題
- 水筒
- 体操服
- エプロン
- 提出プリント
- 財布
- 定期入れ
- ハンカチ

## 4-4. D-04 明日の登校予定

### 選択肢

- 1時間目から
- 2時間目から
- 3時間目から
- 給食から
- 午後から
- 明日の朝に決める
- 休む予定
- その他

選択結果から、翌朝の声かけ予定を逆算する。

## 4-5. D-05 19時（夕食後）の振り返り

### 通常モード

```text
今日のことを一緒に確認しよう

[できたこと]
[手伝ってもらったこと]
[困ったこと]
[明日に回すこと]
[明日の予定]
```

### 夏休みモード

```text
明日の予定を一緒に確認しよう

[外出]
[学習]
[休憩]
[必要な物]
[起きる時刻]
```

### 完了時の処理

- 夜タスクを再生成する。
- 翌朝タスクを生成する。
- 母用ホームへ「確認完了」を表示する。

# 5. 母用ホーム設計

## 5-1. M-01 母用ホーム

### ヘッダー

- 画面名: 声かけリマインダー
- 日付切替
- 通知アイコン
- 設定アイコン

### 時間帯タブ

```text
[ ☀ 朝 ] [ 🏠 夕方 ] [ 🌙 夜 ]
```

現在時刻に対応するタブを初期表示する。

### タスクカード

```text
┌────────────────────────────┐
│ 🪥 歯磨き         声かけ 2回 │
│ 最新 8:12       次回 8:20    │
│ 「歯磨き、今する？5分後？」  │
│ [声かけ済み] [5分後] [詳細]  │
└────────────────────────────┘
```

### カード表示項目

- アイコン
- タスク名
- 声かけ回数
- 最新の声かけ時刻
- 次回通知時刻
- 完了状態
- 推奨文
- 声かけ済み
- 再通知
- 詳細

### カード操作

- `声かけ済み`: 声かけイベントを即時登録
- `5分後`: 5分後に再通知
- `詳細`: 履歴と声かけ候補を表示
- 完了アイコン: 行動結果を登録

## 5-2. タブごとの初期タスク

### 朝

| タスク | 初期声かけ例 |
|---|---|
| 起床 | おはよう。そろそろ起きる時間だよ |
| 朝食 | 朝ごはん、少し食べる？ |
| 歯磨き | 歯磨き、今する？ 5分後にする？ |
| 服薬 | お薬の時間だよ |
| 着替え | 何時ごろ着替える？ |
| 日焼け止め | 出る前に日焼け止めを塗ろうか |
| 持ち物 | 持ち物、一緒に確認しようか |
| 出発 | そろそろ出発する時間だよ |

### 夕方

| タスク | 初期声かけ例 |
|---|---|
| 帰宅確認 | おかえり。まずは休む？ |
| 今日の予定 | 今日することを確認しようか |
| 宿題 | 宿題は何時ごろする？ |
| 入浴 | お風呂、今にする？ あとにする？ |
| 夕食 | ごはん、そろそろにする？ |
| 明日の持ち物 | 明日いる物を確認しようか |

### 夜

| タスク | 初期声かけ例 |
|---|---|
| 19時（夕食後）の振り返り | 今日のことを一緒に確認しよう |
| 明日の予定 | 明日は何時間目から行く？ |
| 明日の準備 | 明日使う物をカバンに入れようか |
| 歯磨き | 歯磨き、今する？ |
| 服薬 | 夜のお薬の時間だよ |
| スマホの区切り | 今していること、どこで区切る？ |
| 就寝 | そろそろ寝る準備にしようか |
| おやすみ | 今日の声かけはここまで。おやすみ |

# 6. M-02 タスク詳細画面

```text
歯磨き

今日の声かけ: 2回
1回目 7:55
2回目 8:12

おすすめの言い方
● 歯磨き、今する？ 5分後にする？
○ 歯磨きの時間だよ
○ 次は歯磨きにしようか

[この文で声かけ済み]
[文を編集]
[5分後に通知]

行動結果
[完了] [一部] [一緒に] [代行] [後日] [未確認]
```

### 声かけ候補の優先ロジック

1. 母が「今後優先」を設定した文
2. 同じタスクで最近使用した文
3. 声かけ回数に応じた文
4. 標準テンプレート

### 回数別の文例

- 1回目: 穏やかな予告・提案
- 2回目: 時刻や見通しを短く伝える
- 3回目以降: 選択肢を減らし、母の限界を責めずに区切る

例:

- 1回目: 「そろそろ歯磨きの時間だよ」
- 2回目: 「今する？ 5分後にする？」
- 3回目: 「今日は、今するか朝に回すか決めよう」

# 7. 声かけイベント設計

## 7-1. 記録タイミング

`声かけ済み`押下時に、バックグラウンドで保存する。

## 7-2. 保存項目

| 項目 | 型 | 必須 | 説明 |
|---|---|---|---|
| id | UUID | ○ | イベントID |
| household_id | UUID | ○ | 世帯ID |
| daily_task_id | UUID | ○ | 当日のタスク |
| prompted_at | datetime | ○ | 声かけ時刻 |
| prompt_order | integer | ○ | 当日の何回目か |
| prompt_text | text | ○ | 使用した文 |
| source | enum | ○ | template / edited / custom |
| created_by | UUID | ○ | 母ユーザー |
| note | text |  | 任意メモ |

## 7-3. 回数の算出

当日・同一タスクに紐づく `prompt_event` 件数を表示する。

削除したイベントは集計対象外とする。誤操作取り消しは10秒以内ならワンタップで可能とする。

# 8. 完了イベント設計

声かけイベントと分離する。

| 項目 | 型 | 説明 |
|---|---|---|
| id | UUID | 完了イベントID |
| daily_task_id | UUID | 対象タスク |
| status | enum | completed / partial / together / parent_done / deferred / unknown |
| completed_at | datetime | 完了時刻 |
| note | text | 任意メモ |

一つのタスクに複数の声かけイベントと、最大一つの最新完了状態を持たせる。

# 9. 通知設計

## 9-1. 通知内容

```text
8:20　登校準備の声かけタイミングです
「持ち物、一緒に確認しようか」
```

## 9-2. 通知操作

- 声かけ済み
- 5分後
- 10分後
- アプリを開く
- 今日は通知しない

## 9-3. 再通知ルール

- 自動連続通知はしない。
- 母が再通知を選んだ場合のみ設定する。
- タスクが完了した場合、予定済み通知を解除する。
- 時間帯が終了した場合、未完了タスクを次の時間帯へ自動移動しない。母に選択を求める。

# 11. 自動タスク生成ロジック

## 11-1. 登校予定からの逆算

設定値例:

- 家を出るまでの標準準備時間: 60分
- 起床から朝食まで: 15分
- 朝食: 20分
- 歯磨き・服薬: 10分
- 持ち物確認: 10分
- 予備: 5分

`school_start_period` と学校までの移動時間から、各タスク時刻を算出する。

母は自動生成後に個別修正できる。

## 11-2. 今日することからの生成

- 宿題を選択 → 夕方に宿題タスク
- 入浴を選択 → 夕方に入浴タスク
- 提出物を登録 → 夜に持ち物確認タスク
- USJモード → 日焼け止め、財布、定期、ハンカチを追加

## 11-3. 19時（夕食後）の振り返り後

- 翌日の登校予定に変更があれば朝タスクを再計算
- 明日に必要な物を夜の準備タスクへ反映
- 明日に回したことを翌日の計画へ移す

# 12. レポート設計

## 12-1. 日次レポート

```text
7/15（火）

朝
起床      声かけ2回  7:30 / 7:45  完了8:00
朝食      声かけ2回  8:15 / 8:18  一緒に完了
歯磨き    声かけ0回  未確認

夜
スマホ    声かけ4回  20:30 / 20:45 / 21:15 / 22:00
歯磨き    声かけ3回  22:00 / 22:20 / 22:25  完了22:27
```

## 12-2. 週次レポート

- タスク別平均声かけ回数
- 声かけから完了までの平均時間
- 母が代行した回数
- 後日に変更した回数
- 母の負担感

支援者向け表示では、娘の評価ではなく、必要だった支援量として表現する。

# 13. 状態遷移

## 13-1. 日次タスク

```text
scheduled
  ├─ prompt recorded → prompted
  ├─ snooze → scheduled
  ├─ completion → completed / partial / together / parent_done
  ├─ defer → deferred
  └─ end of day → unknown
```

複数回の声かけでは、状態は `prompted` のまま、イベントが追加される。

## 13-2. 19時（夕食後）の振り返り

```text
not_started → in_progress → completed
                      └→ paused
```

# 14. エラー・例外処理

- 通知が届かなかった場合でも、ホームを開くと予定タスクを表示する。
- 声かけ済みを誤って押した場合、10秒間「取り消す」を表示する。
- 同時に二重押下した場合、同一秒・同一タスクの重複登録を防ぐ。
- 娘が見通しを決めなかった場合、母用画面に「未設定」と表示し、母が仮設定できる。
- 19時（夕食後）の振り返りができなかった日は、翌朝の標準テンプレートを利用する。

# 17. Fable向けデザイン指示

- 娘用と母用で役割を明確に分ける。
- 娘用はやわらかい暖色、母用は落ち着いた寒色を基本とする。
- 母用カードは情報量が多いため、タスク名・回数・最新時刻・主ボタンの視線順を明確にする。
- 「声かけ済み」を最も押しやすい主ボタンとする。
- 回数は評価ではなく記録として中立的に表示する。
- 完了状態は色だけでなくアイコンと文言を併用する。
- 朝・夕方・夜のタブは片手で押しやすい高さにする。
- 娘用ホームは三つの主項目を最上部に配置する。

# 18. Codex / Cursor向け実装注意

- `prompt_events` を正本とし、回数はイベント集計で復元可能にする。
- `daily_tasks.prompt_count` はキャッシュとして扱い、整合性を保つ。
- 声かけ記録と完了記録を同一カラムで管理しない。
- 通知スケジュールは端末ローカル通知を優先し、サーバー同期は補助とする。
- タイムゾーンは `Asia/Tokyo` を既定値とする。
- 画面文言はハードコードせず、テンプレートとして管理する。
- タスク・時間帯・通知時刻・声かけ文をユーザーが変更できる構造にする。
- 将来の複数子ども対応を妨げないよう、全データを `household_id` と対象ユーザーに紐づける。
