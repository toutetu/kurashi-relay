# くらしリレー

## Codex UI・実装設計書 v0.2

### Laravel API＋React SPA版

作成日：2026-07-12

# 1. 共通設計

- モバイルファースト
- タブレットを主利用端末にする
- 同じコンポーネントを端末ごとに再配置する
- 人物イラストを使わない
- アイコンとラベルを併用する
- 主要操作は1～2タップ
- タッチ対象は44px以上
- 主要文字は16px以上
- 200%拡大でも主要操作を可能にする
- 横スクロールを発生させない

# 2. ブレークポイント

```text
Mobile:  0～767px
Tablet: 768～1199px
PC:     1200px以上
```

# 3. 母向け配色

赤・黄・青・白を同程度に使う。

```css
:root {
  --mother-red: #ef767a;
  --mother-red-soft: #fff0f1;
  --mother-yellow: #f3c85b;
  --mother-yellow-soft: #fff8db;
  --mother-blue: #68a7e3;
  --mother-blue-soft: #edf6ff;
  --mother-white: #ffffff;
  --mother-bg: #fffdf9;
  --text-main: #28334a;
  --text-muted: #667085;
  --border: #dce5ef;
}
```

- 白：背景、余白、通常カード
- 青：予定、情報、ナビゲーション
- 黄：補助情報、待機、注意
- 赤：重要な変更、中断、期限
- 赤は警告だけでなく柔らかなピンク寄りにも使用する

# 4. 娘向け配色

```css
:root {

  --daughter-blue-strong: #85c8ef;
```

- 水色ベース

# 5. 共通レイアウト

## スマートフォン

- 1カラム
- 固定または追従ヘッダー
- 下部ナビゲーション
- 現在活動とクイック操作を上部へ置く

## タブレット

- 2カラム
- 横向きを主推奨
- 主要操作と情報が最初の画面に収まることを優先

## PC

- 左サイドバー
- 上部ヘッダー
- 3カラムを基本
- 詳細比較画面では横幅を比較表へ優先配分する

## コンパクトホーム（v1.1）

- PC（1200px以上）は、1段目にクイック活動記録・クイック記録・現在の活動、2段目に次の予定・母の体調と気分・時間の内訳を置く2段・3列のコンパクト構成とする。
- スマートフォン・タブレット（1199px以下）は「記録」「今日」タブで段階的に表示する。初期値と不正な値は「記録」とし、選択状態は `?tab=record` または `?tab=today` とURLクエリに同期する。
- 娘の状態・今日の作戦は `/child-plan`、予定への影響・対応事項は `/summary`、ラストウォーは `/last-war` に分ける。ホームは概要を表示し、詳細は専用画面へ分ける。
- PC左サイドバーとモバイル・タブレットのドロワーには「今日のまとめ」と「ラストウォー」を別項目として表示する。モバイル下部ナビゲーションには追加しない。
- カード密度は `compact` と `regular` を使い分ける。PCホームは `compact` とし、操作領域は密度にかかわらず44px以上を保つ。
- 主要操作は共通 `Button` を使用する。`QuickActionButton` と `ScoreControl` は `Button` を内部で使い、アイコンとラベルを併用する。
- 操作可能な要素には、通常時の控えめな影、hover時の最大1pxの浮き、active時の2px下移動・0.99倍・小さい影による押下フィードバックを付ける。disabled時は動かさず、フォーカスリングを維持する。

# 7. 今日の予定と実績

本アプリで特に重要な画面。

## タブレット

```text
時刻 │ 予定 │ 実績
     └──── 差分・原因
```

## PC

```text
時刻 │ 予定 │ 実績 │ 差分・原因
```

## スマートフォン

```text
時刻
予定
実績
差分・原因
```

## 予定欄

- Googleカレンダー由来
- 手入力予定
- 予定なし

## 実績欄

- 活動
- 娘への支援
- 待機・拘束
- 回復
- 実績記録なし

## 差分欄

- 予定どおり
- 開始遅れ
- 実施時間の短縮／延長
- 中断
- 中止
- 延期
- 夜へ振り替え
- 予定外の活動
- 原因
- 失われた時間

予定がない時間を空白表示しない。

# 8. 娘の希望・今日の作戦

項目：

1. 今日どうしたい？
2. 最初に試すこと
3. 大人にしてほしいこと
4. 難しい場合の別案
5. 自信の程度
6. 今日の作戦メモ

禁止する表現：

- 失敗
- 未達成
- 達成率
- 頑張り不足

# 9. 状態表示

- ローディング：スケルトンまたは明確な読み込み表示
- 再取得中：既存表示を残しつつ小さな更新表示
- エラー：内容、再試行ボタン
- データなし：正常な空状態として説明
- 予定なし：予定が登録されていない正常状態
- 実績記録なし：まだ実績が入力されていない状態

通信失敗と正常な空状態を同じ見た目にしない。
